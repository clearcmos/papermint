## 🚨 Emergency Procedures

### 🆘 Critical System Failures

**System Won't Boot**:
```bash
# Boot from NixOS recovery
# Select previous generation from GRUB
# Or use rescue environment

# Check file system
fsck /dev/sdX

# Mount and chroot if needed
mount /dev/sdX /mnt
nixos-enter

# Rollback to previous generation
nixos-rebuild switch --rollback
```

**Complete System Freeze**:
```bash
# Force reboot (as last resort)
echo 1 > /proc/sys/kernel/sysrq
echo b > /proc/sysrq-trigger

# Or hardware reset
# Check logs after reboot:
journalctl --boot=-1 | tail -100
```

**Authentication System Down**:
```bash
# Emergency SSH access (if available)
ssh -i /path/to/emergency-key admin@host

# Check Keycloak service
systemctl status keycloak
journalctl -u keycloak -n 50

# Check OAuth2-proxy services
systemctl status 'oauth2-proxy-*'

# Emergency bypass (if needed)
# Temporarily disable OAuth2-proxy service for specific service
systemctl stop oauth2-proxy-<service>
```

### 🔧 Service Recovery

**Critical Service Failure**:
```bash
# Check service status
systemctl status <service-name>

# View recent logs (safely limited)
journalctl -u <service-name> -n 50

# Emergency restart
systemctl restart <service-name>

# If restart fails, check dependencies
systemctl list-dependencies <service-name> --failed
```

**Database Corruption**:
```bash
# Stop dependent services
systemctl stop <service-using-db>

# Backup current state
cp -r /var/lib/<service> /tmp/backup-$(date +%Y%m%d)

# Restore from backup
restore-backup.sh --service <service> --date latest

# Verify integrity
<service>-integrity-check.sh
```

**Service Unreachable**:
```bash
# Check Traefik status
```

```bash
journalctl -u <service-name> -n 50 | grep -i error
```

**Cockpit Not Accessible**:
```bash
# Check Cockpit service
systemctl status cockpit.socket
systemctl status cockpit

# Check firewall rules
firewall-cmd --list-ports
```

```bash
systemctl status glances

# Check resource usage
top -p $(pgrep glances)

systemctl stop glances
rm -rf /tmp/glances-*
systemctl start glances
```

## 💻 System Issues

### 🖥️ Desktop Environment Problems

**KDE Won't Start**:
```bash
# Check X11/Wayland session
loginctl session-status
echo $XDG_SESSION_TYPE

# Check display manager
systemctl status sddm

# Check for DRM permission errors (SDDM/getty conflict)
journalctl -u display-manager -n 50 | grep -i "permission denied"
# If found, see: issue-2025-12-25-sddm-getty-vt1-conflict.md

# Check if getty is conflicting on VT1
systemctl status getty@tty1.service

# Reset KDE configuration
mv ~/.config/plasma* ~/.config/plasma-backup-$(date +%Y%m%d)
kbuildsycoca5 --noincremental
```

**Slow Login/Performance**:
```bash
# Check login time
systemd-analyze blame | head -20

# Check for conflicting processes
ps aux | grep -E "(kwin|plasmashell)" | head -10

# Clear KDE cache
rm -rf ~/.cache/plasma*
kbuildsycoca5 --noincremental
```

```bash
# Check GPU usage
```

## 🌐 Network Problems

### 🔌 Connectivity Issues

**Host Unreachable**:
```bash
# Check network interfaces
ip link show
ip addr show

# Test connectivity
ping -c 4 gateway_ip
ping -c 4 8.8.8.8
ping -c 4 google.com

# Check routing
ip route show
traceroute target_host
```

**DNS Resolution Problems**:
```bash
# Check DNS configuration
cat /etc/resolv.conf
systemd-resolve --status

# Test DNS resolution
nslookup google.com
dig @1.1.1.1 google.com

# Flush DNS cache
systemd-resolve --flush-caches
```

### 🌐 Service Discovery Issues

**mDNS Not Working**:
```bash
# Check Avahi service
systemctl status avahi-daemon

# Test mDNS resolution
avahi-browse -all
avahi-resolve-host-name hostname.local

# Restart Avahi
systemctl restart avahi-daemon
```

**File Sharing Problems**:
```bash
# Check Samba service
systemctl status smbd
systemctl status nmbd

# Test SMB connectivity
smbclient -L //hostname
mount.cifs //hostname/share /mnt/test

# Check firewall
firewall-cmd --list-services | grep samba
```
