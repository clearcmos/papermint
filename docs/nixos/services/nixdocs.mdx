## 🎯 Overview

NixDocs is the centralized documentation site for the entire NixOS infrastructure, providing a beautiful, organized, and easily navigable interface for all technical documentation. It renders the `/etc/nixos/docs/` directory structure as a professional documentation website using [MkDocs Material](https://squidfunk.github.io/mkdocs-material/) theme.

**Service URL**: https://nixdocs.bedrosn.com
**Backend Port**: 3080 (Nginx → Traefik)
**Host**: misc.home.arpa

---

## 🏗️ Architecture

### 📋 Build System

**MkDocs Material via mkdocs-flake**:

The documentation is built using [mkdocs-flake](https://github.com/applicative-systems/mkdocs-flake), a Nix-native MkDocs distribution with Material theme pre-configured.

```nix
# flake.nix input
mkdocs-flake = {
  url = "github:applicative-systems/mkdocs-flake";
  inputs.nixpkgs.follows = "nixpkgs";
};
```

**Build Process** (`modules/services/nixdocs.nix`):
```nix
mkdocsSite = pkgs.runCommand "nixos-infrastructure-docs" {
  buildInputs = [ mkdocsPackage ];
} ''
  # Create build directory with mkdocs.yml and docs/
  mkdir -p build/docs
  cp ${inputs.self}/mkdocs.yml build/
  cp -r ${inputs.self}/docs/* build/docs/

  # Build with MkDocs
  cd build
  mkdocs build -d $out
'';
```

**Configuration**: `/etc/nixos/mkdocs.yml`
- Navigation structure defined declaratively
- Material theme with dark/light toggle
- Full-text search enabled
- Code syntax highlighting with copy button

### 🎨 Theme Features

**Material Design**:
- Dark mode (default) with light mode toggle
- Responsive design for mobile/tablet/desktop
- Navigation tabs at top level
- Expandable sidebar sections
- Built-in search with suggestions and highlighting

**Markdown Extensions**:
- Admonitions (note, warning, tip boxes)
- Code blocks with syntax highlighting
- Tables with proper styling
- Table of contents with permalinks
- Tabbed content blocks

## 🔧 Configuration

### 📄 Service Configuration

**NixOS Module**: `/etc/nixos/modules/services/nixdocs.nix`
```nix
services.nginx.virtualHosts."nixdocs.bedrosn.com" = {
  listen = [ { addr = "127.0.0.1"; port = 3080; } ];
  root = "${mkdocsSite}";

  locations."/" = {
    tryFiles = "$uri $uri/ $uri.html /index.html";
    extraConfig = ''
      # Security headers
      add_header X-Frame-Options "SAMEORIGIN" always;
      add_header X-Content-Type-Options "nosniff" always;

      # Cache static assets
      location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
      }
    '';
  };
};
```

**Traefik Integration**: Defined in `modules/services/traefik.nix`
```nix
nixdocs = {
  rule = "Host(`nixdocs.bedrosn.com`)";
  service = "oauth2-proxy-nixdocs";
  tls = {
    certResolver = "le";
  };
  middlewares = [ "enhanced-security-headers" "strip-server-headers" ];
};
```

### 🔐 Authentication

**OAuth2-Proxy Configuration**: Protected via `modules/services/oauth2-proxy-nixdocs.nix`
- **Port**: 4189 (OAuth2-proxy) → 3080 (Nginx backend)
- **Group**: `nixdocs-access` in Keycloak
- **Identity Provider**: Keycloak with Google SSO
- **Session Duration**: 4 hours with 15-minute refresh

**Access Control**:
- **Authentication**: OAuth2-proxy with Keycloak (Google SSO)
- **Authorization**: Group-based via Keycloak (`nixdocs-access` group)
- **Security Headers**: Enhanced CSP, HSTS, X-Frame-Options via Traefik middlewares
- **HTTPS Only**: TLS termination via Traefik with Let's Encrypt

## 🚀 Deployment

### 📦 Deployment Process

**Automatic Deployment**:
1. **Edit Docs**: Modify markdown files in `/etc/nixos/docs/`
2. **Update Nav**: Add new pages to `/etc/nixos/mkdocs.yml` if needed
3. **Git Add**: `git add` new files (critical for flake inclusion)
4. **Build**: `nixos-rebuild switch --flake .#misc --target-host misc.home.arpa`
5. **Verify**: Site automatically rebuilds and redeploys

**Manual Validation**:
```bash
# Check service status
ssh misc.home.arpa "systemctl status nginx"

# Test site accessibility (should redirect to OAuth2)
curl -I https://nixdocs.bedrosn.com
```

### 🔄 Adding New Documentation

**Adding a New Page**:
1. Create the markdown file in appropriate `/etc/nixos/docs/` subdirectory
2. Add entry to `nav:` section in `/etc/nixos/mkdocs.yml`
3. `git add` the new file
4. Rebuild misc

**Adding a New Section**:
1. Create new directory in `/etc/nixos/docs/`
2. Add `overview.md` as section index
3. Add section to `nav:` in mkdocs.yml
4. Rebuild misc
