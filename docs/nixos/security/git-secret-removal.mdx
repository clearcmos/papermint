## 🎯 Overview

This guide covers the process of detecting, removing, and preventing secrets from being committed to git repositories. It includes procedures for scanning repositories, removing secrets **from ALL of git history** (every commit ever made), and setting up automated scanning.

**What This Removes**:
- ✅ Secrets in **current files** (working directory)
- ✅ Secrets in **all commits** (entire git history)
- ✅ Secrets in **all branches** (including old branches)
- ✅ Secrets that were **previously deleted** but still in history

**Use Cases**:
- Accidentally committed API keys or tokens
- Hardcoded passwords in configuration files
- OAuth secrets or encryption keys in code
- Personal information (emails, usernames, etc.)
- Any sensitive data that made it into git history

**⚠️ Critical**:
- This **rewrites ALL git history** - every commit SHA will change
- Requires **force-pushing** to remote repository
- Anyone with clones must re-clone or `git pull --force`
- Creates a complete backup before starting

## 🔍 Secret Detection

### 🐷 TruffleHog Commands

**Basic Scanning**:
```bash
# Scan entire git history (all results including unverified)
thog

# Scan current filesystem only (with filtering)
thog-all

# Run full TruffleHog with custom options
trufflehog git file://. --results=verified
trufflehog filesystem . --json
```

**Understanding Results**:
```
Found unverified result 🐷🔑❓
Detector Type: TwitchAccessToken
Decoder Type: PLAIN
Raw result: glv6u71u1epnahg2m84dgotq8sp7sb
Commit: 0ba0602ad3d803f2e29295056d67fa4015efbf71
Email: clearcmos <whatever@domain.com>
File: modules/desktop/twitch-tui.nix
Line: 26
Repository: git@github.com:clearcmos/nixos.git
Timestamp: 2025-10-20 22:53:42 +0000
```

**Key Fields**:
- **Detector Type**: What kind of secret (API key, token, password)
- **Raw result**: The actual secret value found
- **Commit**: Exact git commit SHA containing the secret
- **File**: Path to file containing the secret
- **Line**: Line number where secret appears
- **Verification issue**: Why TruffleHog couldn't verify it (may be expired/invalid)

### 🔎 Multi-Repository Scanning

**Scan All Repositories**:
```bash
# Scan /etc/nixos and all user repos (requires root)
gscan

# Output shows:
# - Total repositories scanned
# - Repositories with secrets detected
# - Clean repositories
```

**How gscan Works**:
1. Scans `/etc/nixos` if it's a git repository
2. Scans all repositories in `~/git/mine` and `~/git/forked` matching `clearcmos` remote
3. Filters out known false positives from `/etc/truffleignore`
4. Reports summary of findings

## 🧹 Secret Removal Process

### 📋 Prerequisites

**Before Starting**:
- [ ] Commit and push all current work
- [ ] Verify you have `git-filter-repo` available (`nix-shell -p git-filter-repo`)
- [ ] Know the exact secret string to remove
- [ ] Understand that this rewrites git history (force push required)
- [ ] Notify anyone with repository clones about upcoming history rewrite

### 🔥 Step-by-Step Removal

**Step 1: Commit Current Work**
```bash
cd /etc/nixos

# Check status
git status

# Commit any pending changes
git add .
git commit -m "feat: current work before history rewrite"
git push origin main
```

**Step 2: Create Backup Branch**
```bash
# Create local backup
git branch backup-before-filter-$(date +%Y%m%d)

# Verify backup exists
git branch | grep backup
```

**Step 3: Create Replacement File**
```bash
# Create mapping file (replace SECRET_VALUE with actual secret)
cat > /tmp/secret-replace.txt <<'EOF'
SECRET_VALUE==>***REMOVED***
EOF

# Verify the file
cat /tmp/secret-replace.txt
```

**Example for our Twitch token removal**:
```bash
cat > /tmp/token-replace.txt <<'EOF'
glv6u71u1epnahg2m84dgotq8sp7sb==>***REMOVED***
EOF
```

**Step 4: Run git-filter-repo**
```bash
# Remove any previous filter-repo state
rm -rf .git/filter-repo

# Run the filter (this rewrites ALL commits)
nix-shell -p git-filter-repo --run "git filter-repo --replace-text /tmp/secret-replace.txt --force"

# Expected output:
# - Notice about removing 'origin' remote (normal behavior)
# - Progress: "Parsed XXX commits"
# - Completion: "Completely finished after X.XX seconds"
```

**What Happens**:
- Every commit in history is rewritten
- Secret is replaced with `***REMOVED***` everywhere it appears
- All commit SHAs change (history is rewritten)
- Origin remote is removed (must be re-added)

**Step 5: Verify Removal**
```bash
# Scan with TruffleHog
thog

# Expected: No secrets found, or only remaining ones (not the removed secret)

# Search git history directly
git log --all -S "SECRET_VALUE" --oneline

# Expected: Empty output (no commits contain the secret)

# Check old commit that had the secret
git show COMMIT_SHA:path/to/file.ext | grep "SECRET"

# Expected: Shows ***REMOVED*** instead of actual secret
```

**Step 6: Force Push to Remote**
```bash
# Re-add origin remote (git-filter-repo removes it)
git remote add origin git@github.com:USERNAME/REPO.git

# Verify remote
git remote -v

# Force push cleaned history
git push --force origin main

# Expected output:
# + oldSHA...newSHA main -> main (forced update)
```

**Step 7: Cleanup**
```bash
# Optionally delete backup branch
git branch -D backup-before-filter-YYYYMMDD

# Clean up temporary files
rm /tmp/secret-replace.txt
```

```bash
# Run complete scan
```

**Verify Remote History**:
```bash

git fetch origin

# Search remote history
```

**Check Remote Commit**:
```bash

```

- Navigate to GitHub/GitLab repository
- Check commit history
- View old commits where secret existed
- Verify: Shows `***REMOVED***` instead of actual secret

## 🛡️ Prevention

### 🔒 Pre-Commit Scanning

**TruffleHog Wrapper Configuration**:

The system includes automatic filtering for git scans in `/etc/nixos/modules/core/security/trufflehog.nix`:

```nix
# Wrapper filters git scans through ignore patterns
# Removes entire finding blocks when raw results match /etc/nixos/templates/trufflehog-ignore-strings.txt
```

**Ignore Patterns** (`/etc/nixos/templates/trufflehog-ignore-strings.txt`):
```
# Add known false positives (NOT real secrets!)
# One pattern per line

# Example proxy URL from documentation
http://user:password@proxy

# Add other false positives here (documentation examples, placeholders, etc.)
```

**⚠️ Important**: Only add **false positives** to ignore list, never real secrets!

### 🚨 Automated Scanning

**Pre-Push Scanning**:

The `gscan` command (part of `gpush` workflow) automatically scans before pushing:

```bash
# Scan all repositories before pushing
gscan

# Exit codes:
# 0 = All clean
# 1 = Secrets found (prevents push)
```

**Integration with Git Workflow**:
```bash
# Smart commit and push (includes scanning)
gpush --all   # All repos
gpush --here  # Current repo only
```

### 📝 Best Practices

**DO**:
- ✅ Store secrets in agenix encrypted files
- ✅ Use environment variables for sensitive data
- ✅ Run `thog` before committing changes
- ✅ Use `.gitignore` for sensitive file patterns
- ✅ Review diffs before committing
- ✅ Scan repositories regularly with `gscan`

**DON'T**:
- ❌ Hardcode API keys or tokens in config files
- ❌ Commit `.env` files with real credentials
- ❌ Add real secrets to ignore patterns
- ❌ Assume deleted files are gone from history
- ❌ Share repositories before scanning
- ❌ Ignore TruffleHog warnings

## 🚨 Post-Compromise Actions

### 🔥 If Secret Was Exposed

**Immediate Actions** (within 5 minutes):
1. **Revoke the Secret**: Immediately invalidate at the service provider
2. **Scan History**: Confirm extent of exposure
3. **Remove from Git**: Follow removal process above
4. **Generate New Secret**: Create replacement credential
5. **Update Configuration**: Deploy new secret via agenix

**Example: Twitch Token Compromise**:
```bash
# 1. Revoke at https://www.twitch.tv/settings/connections
# 2. Scan to confirm
thog

# 3. Remove from history (see above)
# 4. Generate new token at Twitch
# 5. Store in agenix
cd /etc/nixos/secrets
echo "NEW_TOKEN_VALUE" | nix run github:ryantm/agenix -- -e twitch-access-token.age -i ../keys/cmos-agenix.key

# 6. Rebuild to deploy
nixos-rebuild switch --flake .#cmos
```

**Follow-Up Actions** (within 24 hours):
- [ ] Audit access logs for suspicious activity
- [ ] Review all commits in affected timeframe
- [ ] Scan other repositories for similar issues
- [ ] Update incident documentation
- [ ] Review and improve prevention measures

### 📊 Impact Assessment

**Questions to Answer**:
- How long was the secret exposed?
- Was the repository public or private?
- Has anyone cloned the repository?
- Is there evidence of unauthorized access?
- Are there related secrets that should be rotated?

## 🔧 Troubleshooting

### ❌ Common Issues

**Issue: git-filter-repo fails with "already_ran" error**
```bash
# Solution: Remove filter-repo state
rm -rf .git/filter-repo

# Then re-run filter-repo command
```

**Issue: Secret still appears in some commits**
```bash
# Cause: Secret has variations (with/without quotes, different encoding)
# Solution: Add all variations to replacement file

cat > /tmp/replace.txt <<'EOF'
"secret-value"==>***REMOVED***
secret-value==>***REMOVED***
'secret-value'==>***REMOVED***
EOF
```

**Issue: Origin remote removed after filter-repo**
```bash
# Expected behavior - just re-add it
git remote add origin git@github.com:user/repo.git
```

**Issue: Force push rejected**
```bash
# Check if branch is protected
# GitHub: Settings -> Branches -> Unprotect temporarily

# Verify you have push permissions
git remote -v

# Try with verbose output
git push --force origin main --verbose
```

### 🔍 Debugging Commands

**Check what git-filter-repo would change**:
```bash
# Dry-run mode (if supported)
git-filter-repo --replace-text /tmp/replace.txt --dry-run

# Or create test repository first
cp -r /etc/nixos /tmp/nixos-test
cd /tmp/nixos-test
# Run filter-repo on test copy
```

**Verify replacement file syntax**:
```bash
# Check file contents
cat /tmp/replace.txt

# Test pattern matching
echo "my-secret-value" | grep -F -f /tmp/replace.txt
```

## 🚀 Automated Secret Removal

### 🔥 `nuke-secret` Command

**Quick automated removal** for removing secrets from current files, all commits, and entire git history:

```bash
# Remove secret from entire repository (files + all history)
nuke-secret "my-secret-string"

# Example: Remove email from everywhere
nuke-secret "olduser@example.com"

# Example: Remove API key
nuke-secret "sk-1234567890abcdef"
```

**What It Does**:
1. ✅ Validates you're in a git repository
2. ✅ Commits any pending changes
3. ✅ Creates backup branch: `backup-nuke-YYYYMMDD-HHMMSS`
4. ✅ Uses `git-filter-repo` to replace secret with `***REMOVED***`
5. ✅ Removes secret from **current files AND all git history**
6. ✅ Verifies removal was successful
7. ✅ Shows instructions for force-pushing to remote

**Interactive Mode**:
- Confirms the secret to remove
- Shows preview of what will be replaced
- Asks for confirmation before proceeding
- Does NOT automatically force push (you control when)

**Verification After Running**:
```bash
# Search current files for the secret
grep -r "SECRET" .

# Search all git history for the secret
git log --all -S "SECRET" --oneline

# Both should return empty results
```

**Force Push After Verification**:
```bash
# Once you've verified the removal worked correctly
git remote add origin git@github.com:user/repo.git  # If removed
git push --force origin main
```

**Safety Features**:
- Creates timestamped backup branch automatically
- Shows clear warnings about history rewrite
- Provides verification commands
- Requires manual force push (prevents accidental damage)
