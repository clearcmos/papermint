## Overview

This document describes how to bootstrap a fresh NixOS installation with this flake-based configuration. The bootstrap process solves the chicken-and-egg problem where secrets are needed to build the system, but the system needs to be built to have access to secret management tools.

## The Chicken-and-Egg Problem

On a fresh NixOS system:
- You need 1Password CLI to fetch secrets
- You need secrets to build the NixOS configuration  
- 1Password CLI gets installed by the NixOS configuration
- **Result**: Can't build without secrets, can't get secrets without building

## Solution: Bootstrap Script

The `scripts/bootstrap-fresh-system.sh` script solves this by:

1. **Temporarily installing** 1Password CLI via `nix-shell`
2. **Prompting for** your service account token
3. **Storing** the token in `/root/.bashrc` 
4. **Restoring** all age keys and encrypted secrets from 1Password
5. **Enabling** successful `nixos-rebuild switch`

## Prerequisites

Before starting, ensure you have:
- ✅ Fresh NixOS installation with basic KDE/desktop
- ✅ Network connectivity
- ✅ Git installed (`nix-shell -p git` if needed)
- ✅ Your 1Password service account token (starts with `ops_`)

## Bootstrap Procedure

### Step 1: Clone Repository

```bash
# As regular user
cd /tmp
git clone <your-repo-url> nixos-config
cd nixos-config

# Move to system location (as root)
sudo mv /tmp/nixos-config /etc/nixos
cd /etc/nixos
```

### Step 2: Run Bootstrap Script

```bash
# Must be run as root
sudo scripts/bootstrap-fresh-system.sh
```

The script will:
1. Temporarily install 1Password CLI
2. Prompt you for your service account token
3. Test 1Password connectivity
4. Restore all age keys (cmos.key, misc.key, jimmich.key, t2.key)
5. Restore encrypted secrets (core.yaml)
6. Restore secrets configuration

### Step 3: Build Your Host

After bootstrap completion, rebuild for your specific host:

```bash
# For desktop workstation
sudo nixos-rebuild switch --flake .#cmos

# For server hosts
sudo nixos-rebuild switch --flake .#misc
sudo nixos-rebuild switch --flake .#jimmich
```

### Step 4: Verification

After successful rebuild:

```bash
# Verify 1Password CLI is now permanently available
op --version

# Verify secrets are accessible
ls /run/agenix/

# Test secret decryption
nix run github:ryantm/agenix -- -d secrets/nicholas-password.age -i keys/$(hostname)-agenix.key
```

## Multi-Host Support

The bootstrap process works for **all hosts** (cmos, misc, jimmich, t2):

- ✅ **Restores ALL age keys**: Not just current host
- ✅ **Shared secrets**: Core secrets work across all hosts  
- ✅ **Host flexibility**: Can bootstrap any host and rebuild any other host
- ✅ **Cross-host networking**: All hosts can mount each other's shares

Example multi-host workflow:
```bash
# Bootstrap on host A (e.g., cmos)
sudo scripts/bootstrap-fresh-system.sh

# Can now rebuild any host from this location
sudo nixos-rebuild switch --flake .#cmos  # Local host
sudo nixos-rebuild switch --flake .#misc --target-host misc.home.arpa  # Remote host
```

## What Gets Restored

### Age Keys (`keys/`)
- `cmos.key` - Desktop workstation decryption key
- `misc.key` - Server host decryption key  
- `jimmich.key` - Server host decryption key

### Secrets (`secrets/`)
- `core.yaml` - Encrypted secrets shared by all hosts
  - SSH keys, CIFS credentials, network configuration

### Configuration
- `secrets.nix` - Agenix configuration with all host public keys

## Troubleshooting

### "1Password CLI installation failed"
```bash
# Test nix-shell access
nix-shell -p _1password-cli --run "op --version"
```

### "Authentication failed"
```bash
# Check token format (should start with 'ops_')
echo $OP_SERVICE_ACCOUNT_TOKEN | cut -c1-4

# Test token manually
export OP_SERVICE_ACCOUNT_TOKEN="your-token-here"
nix-shell -p _1password-cli --run "op vault list"
```

### "No age keys found in 1Password"
```bash
# List items in vault
nix-shell -p _1password-cli --run "op item list --vault=nixos"

# Look for items named: cmos-agenix-key, misc-agenix-key, jimmich-agenix-key, t2-agenix-key
```

### "Agenix decryption fails after restore"
```bash
# Verify age key exists and has correct permissions
ls -la keys/
chmod 600 keys/*.key

# Test decryption manually
nix run github:ryantm/agenix -- -d secrets/nicholas-password.age -i keys/$(hostname)-agenix.key
```

## Security Considerations

### Service Account Token
- ✅ **Read-only access**: Service account has minimal required permissions
- ✅ **Stored securely**: Token goes in `/root/.bashrc` (root-only access)
- ⚠️ **Backup token**: Keep a copy of your service account token in 1Password

### Age Keys
- ✅ **Proper permissions**: Restored with 600 permissions (owner-only)
- ✅ **Host-specific**: Each host has its own decryption key
- ✅ **Backed up**: Keys are stored in 1Password for disaster recovery

### Network Security
- ✅ **HTTPS only**: All 1Password API calls use HTTPS
- ✅ **Temporary access**: Bootstrap only installs CLI temporarily
- ✅ **No plaintext secrets**: Secrets remain encrypted during transfer

## Post-Bootstrap Workflow

After successful bootstrap, normal operations resume:

### Regular Rebuilds
```bash
# Standard rebuild (includes secret sync)
sudo nixos-rebuild switch --flake .#<host>
```

### Secret Management
```bash
# Edit encrypted secrets
nix run github:ryantm/agenix -- -e secrets/secret-name.age -i keys/$(hostname)-agenix.key

# Backup to 1Password (manual process required)
```

### Adding New Hosts
1. Generate new age key: `age-keygen -o keys/newhostname-agenix.key`
2. Add public key to `secrets/secrets.nix`
3. Re-encrypt shared secrets with new key included
4. Backup new key to 1Password manually
5. Bootstrap new host with updated 1Password vault

## Alternative Bootstrap Methods

### Manual Method (if script fails)
```bash
# 1. Install 1Password CLI temporarily
nix-shell -p _1password-cli

# 2. Set up authentication
export OP_SERVICE_ACCOUNT_TOKEN="your-token"
echo 'export OP_SERVICE_ACCOUNT_TOKEN="your-token"' >> /root/.bashrc

# 3. Create directories
mkdir -p keys secrets

# 4. Restore age keys manually
op item get "cmos-agenix-key" --vault="nixos" --field="credential" --reveal > keys/cmos-agenix.key
op item get "misc-agenix-key" --vault="nixos" --field="credential" --reveal > keys/misc-agenix.key
op item get "jimmich-agenix-key" --vault="nixos" --field="credential" --reveal > keys/jimmich-agenix.key
op item get "t2-agenix-key" --vault="nixos" --field="credential" --reveal > keys/t2-agenix.key
chmod 600 keys/*.key

# 5. Restore encrypted secrets from 1Password
# Each secret needs to be restored individually from 1Password
# and re-encrypted with agenix
mv "/tmp/core.yaml" "secrets/core.yaml"

# 7. Rebuild
nixos-rebuild switch --flake .#<host>
```
