## Overview

Agenix is an age-based secret management system for NixOS that encrypts secrets at rest and decrypts them during system activation. Unlike sops-nix (too complex), agenix provides a simple, reliable solution that works at NixOS activation time.

### Key Features
- **Age encryption**: Modern, simple file encryption
- **Multi-host support**: Each host can be self-sufficient
- **Evaluation-time compatibility**: Secrets available when NixOS needs them
- **Declarative configuration**: All secrets defined in Nix
- **Git-safe**: Only encrypted files are committed

### Hosts Supported
- **cmos**: Desktop workstation (192.168.1.2)
- **misc**: Server host (192.168.1.5) 
- **jimmich**: Server host (192.168.1.10)
- **t2**: MacBook Air T2 workstation (192.168.1.80)

## Why Agenix?

We chose agenix for its simplicity and reliability:

- **Simple architecture**: Age-based encryption, no complex GPG setup
- **NixOS-native**: Works seamlessly with NixOS activation model
- **Multi-host support**: Each host can independently decrypt its secrets
- **Evaluation-time compatible**: Secrets available when NixOS needs them
- **Git-friendly**: Only encrypted files are committed

## Architecture

### Secret Categories

**Shared Secrets** (decryptable by all hosts):
- User passwords and hashes
- CIFS/Samba credentials  
- Database passwords
- Application secrets

**Host-Specific Secrets** (decryptable only by that host):
- SSH private keys (root and user)
- Host-specific certificates
- Host-specific tokens

### Configuration Files

**secrets.nix** - Defines encryption rules:
```nix
let
  # Host age keys  
  cmosKey = "age13x2a9ecnqa8g2ymqq7fp2cpna6axegy82gnujcfp93scrzgr0qks8vctz3";
  miscKey = "age1shnfssh3qe5h7duc3ya30sl7dehznq49m0jepdwjry83mamzlpwsufaedm";
  jimmichKey = "age1thqyhu2h6ehlk80slmhceshzrl0n9fnk30qf2zuj6llzx6uzdsmsesn8rm";
  t2Key = "age16k6qw9qxk666yh6kv7ls8z33ywtz8hx74ddfcl4fysn4mrjhwy6sg07ymt";
  allKeys = [ cmosKey miscKey jimmichKey t2Key ];
  
in {
  # Shared secrets - all hosts can decrypt
  "nicholas-password-hash.age".publicKeys = allKeys;
  "cifs-username.age".publicKeys = allKeys;
  "cifs-password.age".publicKeys = allKeys;
  "keycloak-postgres-password.age".publicKeys = allKeys;
  
  # Host-specific SSH identity keys - only that host can decrypt
  "cmos-root-private-key.age".publicKeys = [ cmosKey ];
  "cmos-nicholas-private-key.age".publicKeys = [ cmosKey ];
  "misc-root-private-key.age".publicKeys = [ miscKey ];
  "misc-nicholas-private-key.age".publicKeys = [ miscKey ];
  "jimmich-root-private-key.age".publicKeys = [ jimmichKey ];
  "jimmich-nicholas-private-key.age".publicKeys = [ jimmichKey ];
  "t2-root-private-key.age".publicKeys = [ t2Key ];
  "t2-nicholas-private-key.age".publicKeys = [ t2Key ];
}
```

**agenix-secrets.nix** - NixOS module defining secret access:
```nix
{ config, lib, pkgs, ... }:

{
  # Define all encrypted secrets with agenix
  age.secrets = {
    # User credentials
    nicholas-password-hash = {
      file = ../../secrets/nicholas-password-hash.age;
      mode = "0600";
      owner = "root";
      group = "root";
    };
    
    # CIFS credentials  
    cifs-username = {
      file = ../../secrets/cifs-username.age;
      mode = "0600";
      owner = "root";
      group = "root";
    };
    
    # ... (all other secrets)
  };
  
  # Use host-specific age key for decryption
  age.identityPaths = [ "/etc/nixos/keys/${config.networking.hostName}-agenix.key" ];
}
```

## Deployment Process

### Initial Setup (One-time)

1. **Install agenix in flake.nix**:
   ```nix
   inputs = {
     agenix = {
       url = "github:ryantm/agenix";
       inputs.nixpkgs.follows = "nixpkgs";
     };
   };
   
   outputs = { agenix, ... }: {
     nixosConfigurations.hostname = nixpkgs.lib.nixosSystem {
       modules = [
         agenix.nixosModules.default
         ./hosts/hostname/default.nix
       ];
     };
   };
   ```

2. **Generate age keys for each host**:
   ```bash
   cd /etc/nixos/keys
   nix shell nixpkgs#age --command age-keygen -o cmos-agenix.key
   nix shell nixpkgs#age --command age-keygen -o misc-agenix.key  
   nix shell nixpkgs#age --command age-keygen -o jimmich-agenix.key
   nix shell nixpkgs#age --command age-keygen -o t2-agenix.key
   
   # Set proper permissions
   chmod 600 *.key
   ```

3. **Create secrets.nix** with public keys from step 2

4. **Create agenix-secrets.nix** module defining all secrets

5. **Import agenix-secrets.nix** in all host configurations

### Secret Creation Workflow

For each secret that needs to be encrypted:

```bash
cd /etc/nixos/secrets

# Encrypt new secret (interactive editor will open)
nix shell github:ryantm/agenix --command agenix -e secret-name.age -i /etc/nixos/keys/<host>-agenix.key

# Validate decryption works
nix shell github:ryantm/agenix --command agenix -d secret-name.age -i /etc/nixos/keys/<host>-agenix.key
```

**Critical**: 
- Must run from `/etc/nixos/secrets` directory
- `secrets.nix` must exist in `/etc/nixos/` root (agenix requirement)
- **SSH private keys MUST have trailing newline after `-----END OPENSSH PRIVATE KEY-----`**
- **Manual encryption only** - automation fails reliably

## Issues Encountered & Solutions

### Issue 1: SSH Authorized Keys Evaluation-Time Requirement  

**Problem**: NixOS SSH module uses `lib.readFile` on authorized key files during evaluation, but agenix secrets are only available at activation time.

**Error**: `error: access to absolute path '/run' is forbidden in pure evaluation mode`

**Solution**: Hardcode SSH authorized keys directly in configuration instead of using files:
```nix
openssh.authorizedKeys.keys = [
  "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIPD5PMDoa0TPKOxNlw3WNYvEnh4/zctnRPFEiIafg/b9 mba.home.arpa"
  # ... other keys
];
```

### Issue 3: Automated Secret Encryption Failures

**Problem**: Using `EDITOR="bash -c 'echo content > \$1'"` resulted in empty encrypted files.

**Root Cause**: The echo command approach didn't properly write content that agenix could read.

**Solution**: **Manual interactive encryption required**. Automated approaches don't work reliably.

**CRITICAL**: For SSH private keys, always add a trailing newline after pasting the key content.

### Issue 4: SSH Private Key Format Errors

**Problem**: SSH private keys cause "Load key: error in libcrypto" when missing trailing newline.

**Root Cause**: OpenSSH requires a newline character (LF) after the `-----END OPENSSH PRIVATE KEY-----` line.

**Solution**: 
1. When pasting SSH private keys into agenix editor
2. **Always press Enter after the END line** to add trailing newline
3. Save and exit - this fixes libcrypto errors

### Issue 5: Multi-Host Key Re-encryption

**Problem**: When changing `secrets.nix` from single-key to multi-key, existing encrypted files retained old key restrictions.

**Solution**: Re-encrypt all affected secrets by:
1. Opening with `agenix -e` 
2. Making minimal change (add/remove space)
3. Save - triggers re-encryption with new keys from `secrets.nix`

### Issue 6: Git Flake Visibility

**Problem**: `nixos-rebuild` couldn't see new `.age` files and keys.

**Solution**: `git add` all new files before building, even if not ready to commit.

### Issue 7: Age Key Format Issues

**Problem**: Initially tried using SSH keys directly with agenix, got "unknown identity type" errors.

**Solution**: Generate dedicated age keys instead of reusing SSH keys.

## Adding New Secrets

### For Shared Secrets (All Hosts Need Access)

1. **Add to secrets.nix**:
   ```nix
   "new-shared-secret.age".publicKeys = allKeys;
   ```

2. **Add to agenix-secrets.nix**:
   ```nix
   new-shared-secret = {
     file = ../../secrets/new-shared-secret.age;
     mode = "0600";
     owner = "root"; 
     group = "root";
   };
   ```

3. **Encrypt the secret**:
   ```bash
   cd /etc/nixos/secrets
   nix shell github:ryantm/agenix --command agenix -e new-shared-secret.age -i /etc/nixos/keys/cmos-agenix.key
   # Type secret content in editor, save
   ```

4. **Use in configuration**:
   ```nix
   services.myservice.passwordFile = config.age.secrets.new-shared-secret.path;
   ```

### For Host-Specific Secrets

1. **Add to secrets.nix**:
   ```nix
   "hostname-specific-secret.age".publicKeys = [ hostnameKey ];
   ```

2. **Follow same steps as shared secrets** but use the specific host's key for encryption

### For SSH Private Keys

**IMPORTANT**: SSH private keys should be host-specific (only that host can decrypt).

1. **Generate SSH key pair**:
   ```bash
   ssh-keygen -t ed25519 -f /tmp/hostname_user_key -N "" -C "user@hostname.home.arpa"
   ```

2. **Add public key to SSH known_hosts** in `modules/core/ssh.nix`

3. **Encrypt private key**:
   ```bash
   cd /etc/nixos/secrets
   nix shell github:ryantm/agenix --command agenix -e hostname-user-private-key.age -i /etc/nixos/keys/hostname-agenix.key
   # Paste private key content from /tmp/hostname_user_key
   ```

## Bootstrapping New Hosts

### Prerequisites
- Age keys must be distributed to new host
- Host-specific secrets must be created
- Host configuration must import agenix-secrets.nix

### Step-by-Step Bootstrap Process

1. **Generate Age Key for New Host**:
   ```bash
   cd /etc/nixos/keys
   nix shell nixpkgs#age --command age-keygen -o newhostname-agenix.key
   chmod 600 newhostname-agenix.key
   # Note the public key output
   ```

2. **Update secrets.nix** with new host key:
   ```nix
   let
     newHostKey = "age1..."; # from step 1
     allKeys = [ cmosKey miscKey jimmichKey t2Key newHostKey ];
   ```

3. **Re-encrypt ALL shared secrets** (required when adding new host key):
   ```bash
   cd /etc/nixos/secrets
   # For each shared secret, edit without changes to trigger re-encryption:
   nix shell github:ryantm/agenix --command agenix -e nicholas-password-hash.age -i /etc/nixos/keys/cmos-agenix.key
   # Add space, remove space, save - forces re-encryption with new keys
   ```

4. **Create Host-Specific SSH Keys**:
   ```bash
   # Generate SSH keys for new host
   ssh-keygen -t ed25519 -f /tmp/newhostname_root_key -N "" -C "root@newhostname.home.arpa"
   ssh-keygen -t ed25519 -f /tmp/newhostname_nicholas_key -N "" -C "nicholas@newhostname.home.arpa"
   
   # Encrypt private keys
   cd /etc/nixos/secrets
   nix shell github:ryantm/agenix --command agenix -e newhostname-root-private-key.age -i /etc/nixos/keys/newhostname-agenix.key
   nix shell github:ryantm/agenix --command agenix -e newhostname-nicholas-private-key.age -i /etc/nixos/keys/newhostname-agenix.key
   ```

5. **Update configurations**:
   - Add secrets to `secrets.nix` 
   - Add secrets to `agenix-secrets.nix`
   - Add public keys to `modules/core/ssh.nix` known_hosts
   - Create host configuration importing `agenix-secrets.nix`

6. **Copy Age Key to New Host**:
   ```bash
   # Securely copy age key to new host
   scp /etc/nixos/keys/newhostname-agenix.key root@newhostname:/etc/nixos/keys/
   ```

7. **Deploy**:
   ```bash
   nixos-rebuild switch --flake .#newhostname --target-host newhostname
   ```

### Bootstrap Validation

After deploying new host, validate:
```bash
# On the new host:
ls -la /run/agenix/  # Should show decrypted secrets
systemctl status "*samba*" "*cifs*"  # Services should start
```

### Building and Switching

**Local build** (from any host, if it has the right age key):
```bash
cd /etc/nixos
nixos-rebuild switch --flake .#hostname
```

**Remote build** (deploy to another host):
```bash
cd /etc/nixos
nixos-rebuild switch --flake .#hostname --target-host hostname.home.arpa
```

### Pre-deployment Checks

1. **Validate secrets can be decrypted**:
   ```bash
   cd /etc/nixos/secrets
   bash validate-secrets.sh  # Test current host's secrets
   ```

2. **Test build without activation**:
   ```bash
   nixos-rebuild dry-run --flake .#hostname
   ```

3. **Git status** - ensure all needed files are tracked:
   ```bash
   git add secrets/ keys/ modules/core/agenix-secrets.nix
   ```

### Deployment Dependencies

**Build Requirements**:
- Host age key must exist in `/etc/nixos/keys/`
- All referenced `.age` files must be in git or git staging
- `secrets.nix` must define all referenced secrets

**Runtime Requirements**:  
- Target host must have its age key at `/etc/nixos/keys/hostname-agenix.key`
- Target host must have network access (for remote deployments)
- Target host filesystem must have space for decrypted secrets in `/run/agenix/`

### Shared Secret (All Hosts)

1. **Add to secrets.nix**:
   ```bash
   cd /etc/nixos
   # Edit secrets/secrets.nix, add:
   "new-secret.age".publicKeys = allKeys;
   ```

2. **Add to module**:
   ```bash
   # Edit modules/core/agenix-secrets.nix, add:
   new-secret = {
     file = ../../secrets/new-secret.age;
     mode = "0600";
     owner = "root";
     group = "root";
   };
   ```

3. **Encrypt secret**:
   ```bash
   cd /etc/nixos/secrets
   nix shell github:ryantm/agenix --command agenix -e new-secret.age -i /etc/nixos/keys/cmos-agenix.key
   # Enter secret content in editor
   ```

4. **Use in configuration**:
   ```nix
   # In any module:
   services.myapp.passwordFile = config.age.secrets.new-secret.path;
   
   # In systemd service:
   ExecStart = pkgs.writeShellScript "script" ''
     PASSWORD=$(cat ${config.age.secrets.new-secret.path})
     # use $PASSWORD
   '';
   ```

Same process, but use single host key:
```nix
"hostname-secret.age".publicKeys = [ hostKey ];
```

And encrypt with that host's age key:
```bash

```

## Bootstrapping New Deployments

### From Scratch (New Infrastructure)

1. **Generate Age Keys**:
   ```bash
   mkdir -p /etc/nixos/keys
   cd /etc/nixos/keys
   for host in cmos misc jimmich t2; do
     nix shell nixpkgs#age --command age-keygen -o $host-agenix.key
     chmod 600 $host-agenix.key
   done
   ```

2. **Extract Secrets from Backup Source**:
   
   **From 1Password** (if vault exists):
   ```bash
   # List secrets
   op item list --vault nixos
   
   # Extract each secret
   op item get "secret_name" --vault nixos --reveal
   ```
   
   **From Previous System**:
   - Export from previous secret management system
   - Decrypt existing secrets if accessible
   - Manual entry of critical secrets

3. **Create All Encrypted Secrets**:
   - Use the secret content to encrypt each `.age` file
   - Follow the "Adding New Secrets" process for each one

4. **Deploy to Each Host**:
   ```bash
   # Copy age keys to each host first
   scp keys/hostname-agenix.key root@hostname:/etc/nixos/keys/
   
   # Deploy configuration  
   nixos-rebuild switch --flake .#hostname --target-host hostname
   ```

### Disaster Recovery

**If age keys are lost**:
1. Generate new age keys
2. Update `secrets.nix` with new public keys
3. Re-encrypt ALL secrets with new keys
4. Deploy updated configuration

**If encrypted secrets are corrupted**:
1. Re-extract secret values from source (1Password, backup, etc.)
2. Re-encrypt affected secrets
3. Deploy updated configuration

## Troubleshooting

### Common Issues

**"path does not exist" during agenix commands**:
- Ensure running from `/etc/nixos/secrets` directory
- Check that `secrets.nix` exists and contains the secret definition

**"no identity matched any of the recipients"**:
- Wrong age key being used for decryption
- Secret was encrypted for different keys than available
- Age key file corrupted or wrong permissions

**Empty secret content after decryption**:
- Secret was encrypted incorrectly (empty content)
- Re-encrypt the secret with proper content

**"Load key: error in libcrypto" for SSH keys**:
- SSH private key missing trailing newline
- Re-encrypt and add newline after -----END OPENSSH PRIVATE KEY-----
- This is the #1 cause of SSH key failures

**Build errors about missing secret files**:
- Add new `.age` files to git: `git add secrets/`
- NixOS flakes can't see untracked files

### Service Integration Issues

**Services fail to start with secret access errors**:
1. Check secret is defined in `agenix-secrets.nix`
2. Verify secret path in service: `${config.age.secrets.secret-name.path}`
3. Check service runs after secret decryption (remove `after = ["onepassword-secrets.service"]`)

**Permission denied on secret files**:
- Check secret ownership in `agenix-secrets.nix`
- Verify service runs as user that owns secret
- Check secret mode (0600 for sensitive, 0644 for public-ish)

## Security Considerations

### Key Management

- **Age keys are root-owned, 600 permissions**
- **Each host has its own age key** (no shared keys between hosts)
- **Shared secrets encrypted to multiple hosts** for operational flexibility
- **Host-specific secrets encrypted only to that host** for security isolation

### Secret Distribution

- **Shared secrets**: User passwords, CIFS credentials, database passwords
- **Host-specific secrets**: SSH private keys, host certificates
- **Public data**: SSH public keys, known_hosts entries (hardcoded in config)

### Access Control

- **Secrets decrypted to `/run/agenix/`** (tmpfs, cleared on reboot)
- **Secret files have proper ownership** (root, specific users as needed)
- **Services access secrets via paths** (not environment variables)

### Best Practices

1. **Never commit unencrypted secrets** to git
2. **Always validate encryption** before deploying  
3. **Use host-specific keys for SSH identities**
4. **Use shared keys sparingly** (only for truly shared secrets)
5. **Rotate secrets periodically** (re-encrypt with new keys)
6. **Keep age keys secure** (backup separately from config repo)
