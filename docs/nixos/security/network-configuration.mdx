## Overview

This document details the production-hardened Traefik reverse proxy configuration implemented for bedrosn.com infrastructure. The configuration prioritizes security, privacy, and operational excellence while maintaining high availability.

### Key Security Principles
- **Defense in Depth**: Multiple security layers with no single point of failure
- **Privacy by Design**: Minimal data collection and third-party dependencies
- **Zero Trust**: Every request is authenticated and authorized
- **Fail Secure**: System defaults to secure state on failures

## Security Architecture

```
Internet → Cloudflare DNS → Traefik (443/80) → Services
                    ↓
            [Rate Limiting]
            [Security Headers]
            [TLS Termination]
            [OAuth2/Keycloak Auth]
```

## Static Configuration

### Entry Points

```nix
entryPoints = {
  web = {
    address = ":80";
    http.redirections.entrypoint = {
      to = "websecure";
      scheme = "https";
    };
    forwardedHeaders = {
      trustedIPs = ["127.0.0.1/32", "192.168.1.0/24"];
      insecure = false;
    };
  };
  websecure = {
    address = ":443";
    http.tls.certResolver = "le";
    forwardedHeaders = {
      trustedIPs = ["127.0.0.1/32", "192.168.1.0/24"];
      insecure = false;
    };
    transport = {
      respondingTimeouts = {
        readTimeout = "30s";
        writeTimeout = "30s";
        idleTimeout = "90s";
      };
    };
  };
};
```

**Security Features:**
- **Automatic HTTPS redirect**: All HTTP traffic redirected to HTTPS
- **Trusted IP restriction**: Only local network and localhost can set forwarded headers
- **Response timeouts**: Prevents slow loris attacks and resource exhaustion

### TLS Configuration

```nix
tls = {
  options = {
    default = {
      minVersion = "VersionTLS12";
      sniStrict = true;
      cipherSuites = [
        "TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384"
        "TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305"
        "TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256"
      ];
    };
  };
};
```

**Security Features:**
- **TLS 1.2 minimum**: Industry standard with modern cipher support
- **SNI strict**: Prevents certificate enumeration attacks
- **AEAD ciphers only**: Authenticated encryption with perfect forward secrecy

### Server Transport Security

```nix
serversTransport = {
  forwardingTimeouts = {
    dialTimeout = "5s";
    responseHeaderTimeout = "10s";
    idleConnTimeout = "60s";
  };
  maxIdleConnsPerHost = 64;
};
```

**Security Features:**
- **Connection timeouts**: Prevents resource exhaustion from slow backends
- **Connection pooling**: Efficient resource usage with limits

### Access Log Privacy

```nix
accessLog = {
  fields = {
    headers = {
      defaultMode = "drop";
      names = {
        "User-Agent" = "keep";
        "Referer" = "keep";
      };
    };
  };
  bufferingSize = 100;
};
```

**Privacy Features:**
- **Header dropping**: All headers dropped by default except User-Agent and Referer
- **No sensitive data logging**: Cookies, Authorization headers never logged
- **Buffered writes**: Improved performance and log consistency

## Dynamic Configuration

### Security Middlewares

#### Enhanced Security Headers
```yaml
enhanced-security-headers:
  headers:
    contentSecurityPolicy: "default-src 'self' 'unsafe-inline' 'unsafe-eval'; img-src 'self' data: blob:; font-src 'self' data:; style-src 'self' 'unsafe-inline'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; base-uri 'none'; frame-ancestors 'none'"
    referrerPolicy: "no-referrer"
    permissionsPolicy: "geolocation=(), microphone=(), camera=()"
    stsSeconds: 31536000
    stsIncludeSubdomains: true
    stsPreload: true
    frameDeny: true
    contentTypeNosniff: true
```

**Security Features:**
- **CSP**: Prevents XSS and injection attacks while allowing necessary functionality
- **HSTS**: Force HTTPS for 1 year with subdomain inclusion and preload
- **Permission Policy**: Disable unnecessary browser APIs
- **Anti-clickjacking**: X-Frame-Options and frame-ancestors protection

#### Rate Limiting

```yaml
# OAuth2 endpoints (stricter)
oauth2-rate-limit:
  rateLimit:
    average: 20
    burst: 30
    period: 1m

# General authentication (balanced for Glances)
auth-rate-limit:
  rateLimit:
    average: 100
    burst: 200
    period: 1m
```

**Security Features:**
- **Path-specific limits**: Different limits for different risk levels
- **Burst handling**: Allows temporary spikes while preventing sustained abuse
- **Per-IP tracking**: Automatic IP-based rate limiting

#### Request Security

```yaml
request-body-limit:
  buffering:
    maxRequestBodyBytes: 2097152  # 2MB limit

inflight-limit:
  inFlightReq:
    amount: 100
```

**Security Features:**
- **Body size limits**: Prevents resource exhaustion from large payloads
- **Concurrent request limits**: Prevents resource exhaustion from connection floods

## Security Features

### Removed Components

#### Geographic Blocking (Deprecated)
**Reason for removal**: Privacy concerns and limited security benefit
- GeoIP plugin removed completely
- No third-party IP lookups
- Attackers easily bypass with VPNs/proxies
- OAuth2 + email whitelist provides stronger protection

**Migration**: Geographic restrictions moved to Cloudflare edge rules where needed

#### Plugin Dependencies
- Removed experimental plugin dependencies
- Reduced attack surface and startup time
- Eliminated external API dependencies

### Authentication Integration

#### OAuth2 Proxy (Glances)
- **Provider**: Google OAuth2 with PKCE
- **Email whitelist**: Only specific emails allowed
- **Session security**: 4-hour sessions, 15-minute refresh
- **Cookie security**: Secure, HttpOnly, SameSite=Lax, CSRF protection

#### Keycloak (Identity Provider)
- **SSO**: Google IDP federation with First Broker Login
- **Group-based access**: Per-service authorization groups
- **User whitelist**: Pre-created users only (no self-registration)

## Operational Security

### Systemd Hardening

```nix
systemd.services.traefik.serviceConfig = {
  # Security isolation
  NoNewPrivileges = true;
  PrivateTmp = true;
  PrivateDevices = true;
  ProtectSystem = "strict";
  ProtectHome = true;
  ProtectKernelLogs = true;
  ProtectKernelModules = true;
  ProtectControlGroups = true;
  RestrictSUIDSGID = true;
  RestrictNamespaces = true;
  LockPersonality = true;
  MemoryDenyWriteExecute = true;
  SystemCallArchitectures = "native";
  ReadWritePaths = [ "/var/lib/traefik" ];
  UMask = "0077";
};
```

**Security Features:**
- **Process isolation**: Restricted system access and capabilities
- **Memory protection**: No execute permissions on writable memory
- **Filesystem isolation**: Limited write access to necessary directories
- **Capability dropping**: Minimal required privileges

### File Permissions

```nix
systemd.tmpfiles.rules = [
  "d /var/lib/traefik/dynamic 0755 traefik traefik - -"
  "d /var/lib/traefik/plugins-storage 0755 traefik traefik - -"
  "d /var/lib/traefik/plugins-storage/sources 0755 traefik traefik - -"
  "f /var/lib/traefik/acme.json 0600 traefik traefik - -"
];
```

**Security Features:**
- **ACME certificate protection**: 0600 permissions on certificate storage
- **Directory isolation**: Proper ownership and permissions
- **Automatic cleanup**: Systemd tmpfiles management

### Secret Management

- **ACME certificates**: Automatically managed with proper permissions
- **Cloudflare credentials**: Encrypted with agenix, accessed via environment files
- **TLS certificates**: Let's Encrypt with DNS challenge for better security

## Monitoring & Alerting

### Key Metrics to Monitor

1. **Authentication Failures**
   ```bash
   # OAuth2 callback failures
   journalctl -u oauth2-proxy | grep "AuthFailure" | tail -20
   
   # Rate limit triggers
   journalctl -u traefik | grep "429" | tail -20
   ```

2. **Performance Metrics**
   ```bash
   # Response times on OAuth endpoints
   journalctl -u traefik | grep "/oauth2/" | grep -E "[0-9]+ms$"
   
   # Certificate expiry
   curl -vI https://glances.bedrosn.com 2>&1 | grep "expire"
   ```

3. **Security Events**
   ```bash
   # Unusual request patterns
   journalctl -u traefik | grep -E "(403|429|500)" | tail -20
   
   # Certificate renewal events
   journalctl -u traefik | grep -i "acme\|certificate"
   ```

### Recommended Alerts

1. **OAuth callback error rate >2% over 5 minutes**
2. **Rate limit triggers on /oauth2/* exceeding baseline**
3. **Certificate expiry within 30 days**
4. **Service availability &lt;99% over 1 hour**

## Troubleshooting

### Common Issues

#### Rate Limiting Problems
```bash
# Check current rate limit status
journalctl -u traefik -n 50 | grep -E "(429|rate)"

# Temporary rate limit increase (emergency)
# Edit /var/lib/traefik/dynamic/middlewares.yml
# Restart traefik-dynamic-config.service
```

#### TLS Certificate Issues
```bash
# Check certificate status
openssl s_client -connect glances.bedrosn.com:443 -servername glances.bedrosn.com

# Force certificate renewal
systemctl restart traefik.service

# Check ACME logs
journalctl -u traefik | grep -i acme
```

#### Authentication Flow Problems
```bash
# Check OAuth2 proxy connectivity
curl -I http://localhost:4180/oauth2/auth

# Check Keycloak status
curl -I http://localhost:8089/realms/web

# Verify service routing
curl -H "Host: glances.bedrosn.com" http://localhost:4180/
```

### Emergency Procedures

#### Bypass Authentication (Emergency Only)
1. SSH to misc host
2. Edit `/var/lib/traefik/dynamic/dashboard.yml`
3. Remove authentication middleware from affected service
4. Restart `traefik-dynamic-config.service`
5. **Remember to restore authentication after resolving issue**

#### Rate Limit Emergency Increase
1. Edit `/var/lib/traefik/dynamic/middlewares.yml`
2. Temporarily increase limits (e.g., 500/1000)
3. Restart `traefik-dynamic-config.service`
4. Monitor logs and restore limits once issue resolved
