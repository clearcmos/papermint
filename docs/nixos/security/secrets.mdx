## ⚠️ CRITICAL: Always run agenix from /etc/nixos/secrets

**All agenix commands must be run from `/etc/nixos/secrets` directory.**  
The CLI literally imports `./secrets.nix`, so running from anywhere else fails.

## 🚨 IMPORTANT: User Must Generate Secrets

**Claude Code has environment limitations that cause agenix commands to fail or hang.**  
**Always ask the user to run secret generation commands directly.**

**Correct approach:**
1. Claude provides the exact one-liner command
2. User copies and runs the command in their terminal
3. User reports success/failure back to Claude

**Never attempt to run agenix commands in Claude Code sessions.**

**One-liner pattern for safety:**
```bash
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -e secret.age -i ../keys/host-agenix.key )
```

**Preflight guard (copy this before any agenix command):**
```bash
test -f /etc/nixos/secrets/secrets.nix || { echo "Missing /etc/nixos/secrets/secrets.nix - run from /etc/nixos/secrets"; exit 1; }
```

## Adding a New Host (Critical Process)

When adding a new host to the flake, you MUST follow this complete process to avoid secret decryption failures:

### 1. Generate Age Key for New Host
```bash
cd /etc/nixos/keys
nix shell nixpkgs#age --command age-keygen -o newhostname-agenix.key
chmod 600 newhostname-agenix.key
# Note the public key output for step 2
```

### 2. Update secrets.nix with New Host Key
```bash
# Edit /etc/nixos/secrets/secrets.nix
# Add new host key to the keys list:
newHostKey = "age1..."; # from step 1
allKeys = [ cmosKey miscKey jimmichKey t2Key newHostKey ];
```

### 3. **CRITICAL**: Re-encrypt ALL Shared Secrets
This step is required because existing encrypted secrets don't include the new host key:

```bash
# Preflight check
test -f /etc/nixos/secrets/secrets.nix || { echo "Missing secrets.nix"; exit 1; }

# Re-encrypt each shared secret with current host key:
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -e nicholas-password.age -i ../keys/cmos-agenix.key )
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -e nicholas-password-hash.age -i ../keys/cmos-agenix.key )
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -e cifs-password.age -i ../keys/cmos-agenix.key )
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -e cifs-username.age -i ../keys/cmos-agenix.key )
# ... for each shared secret in allKeys
```

### 4. Add SSH Keys to users.nix
```nix
# Edit /etc/nixos/modules/core/users.nix
# Add deployment host keys to both nicholas and root users:
openssh.authorizedKeys.keys = [
  # ... existing keys
  "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAICWApzyGnR1dTPKRy2+5pprZa8W1ICPmTN26Yf10kuCH root@cmos.home.arpa"
];
```

### 5. Copy Age Key to New Host
```bash
# Ensure proper ownership
sudo chown root:root /etc/nixos/keys/newhostname-agenix.key
scp /etc/nixos/keys/newhostname-agenix.key root@newhost:/etc/nixos/keys/
ssh root@newhost "chown root:root /etc/nixos/keys/newhostname-agenix.key"
```

## Common Troubleshooting

### "no identity matched any of the recipients"
This error means the host's age key can't decrypt the secret. Common causes:

1. **Missing re-encryption after adding host**: Secrets encrypted before host was added to `secrets.nix`
   - **Solution**: Re-encrypt affected secrets (see step 3 above)

2. **Wrong key ownership**: Age key file not owned by root:root
   - **Solution**: `chown root:root /etc/nixos/keys/hostname-agenix.key`

3. **Wrong key used for encryption**: Secret encrypted with wrong host key
   - **Solution**: Re-encrypt with correct key

### SSH Key Authentication Issues for Remote Deployment
If `nixos-rebuild --target-host` fails with SSH authentication:

1. **Add deployment host SSH key to target**: Update `users.nix` with deploying host's public key
2. **Check SSH known_hosts**: Ensure both hosts know each other's host keys
3. **Use password temporarily**: Add `--ask-sudo-password` if needed

### File Ownership Issues After rsync
When syncing `/etc/nixos` to remote hosts, ownership may change:

```bash
# On target host, fix ownership:
sudo chown -R root:root /etc/nixos
sudo chmod 600 /etc/nixos/keys/*.key
```

## Common Secret Generation Patterns

### Random Hex Secrets (JWT, session, encryption keys)
```bash
( cd /etc/nixos/secrets && echo "$(openssl rand -hex 32)" | nix run github:ryantm/agenix -- -e secret-name.age -i ../keys/misc-agenix.key )
```

### Random Base64 Secrets (OAuth client secrets)
```bash
( cd /etc/nixos/secrets && echo "$(openssl rand -base64 32)" | nix run github:ryantm/agenix -- -e secret-name.age -i ../keys/misc-agenix.key )
```

### RSA Private Keys (OIDC signing)
```bash
( cd /etc/nixos/secrets && openssl genrsa -out /tmp/rsa_key.pem 4096 && cat /tmp/rsa_key.pem | nix run github:ryantm/agenix -- -e rsa-key.age -i ../keys/misc-agenix.key && rm /tmp/rsa_key.pem )
```

### Argon2 Password Hashes (legacy, rarely needed)
```bash
( cd /etc/nixos/secrets && echo "PASSWORD_HERE" | argon2 $(openssl rand -base64 16) -id -t 3 -m 15 -p 4 -l 32 -e | nix run github:ryantm/agenix -- -e password-hash.age -i ../keys/misc-agenix.key )
```

### Plain Text Secrets (emails, usernames)
```bash
( cd /etc/nixos/secrets && echo "value-here" | nix run github:ryantm/agenix -- -e secret-name.age -i ../keys/misc-agenix.key )
```

### JSON Structured Secrets (complex configurations)
```bash
# Example: Keycloak admin user configuration
( cd /etc/nixos/secrets && cat > /tmp/admin-user.json <<'EOF'
{
  "username": "admin",
  "firstName": "First",
  "lastName": "Last",
  "email": "admin@example.com",
  "password": "secure-password-here",
  "authMethods": ["password"]
}
EOF
cat /tmp/admin-user.json | nix run github:ryantm/agenix -- -e keycloak-admin-user.age -i ../keys/cmos-agenix.key
rm /tmp/admin-user.json
)
```

### Environment File Secrets (KEY=value format)
```bash
# Example: SSH OAuth client credentials
( cd /etc/nixos/secrets && cat > /tmp/ssh-client.env <<'EOF'
SSH_CLIENT_ID=ssh-pam
SSH_CLIENT_SECRET=generated-secret-here
EOF
cat /tmp/ssh-client.env | nix run github:ryantm/agenix -- -e keycloak-ssh-client.age -i ../keys/cmos-agenix.key
rm /tmp/ssh-client.env
)

# Example: Google OAuth credentials for Keycloak IDP
( cd /etc/nixos/secrets && cat > /tmp/google-oauth.env <<'EOF'
GOOGLE_CLIENT_ID=...apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-...
OAUTH2_PROXY_CLIENT_SECRET=REDACTED_OAUTH2_CLIENT_SECRET
EOF
cat /tmp/google-oauth.env | nix run github:ryantm/agenix -- -e google-oauth-keycloak.age -i ../keys/misc-agenix.key
rm /tmp/google-oauth.env
)
```

**Note:** Replace `secret-name.age`, `misc-agenix.key`, and values as needed.

### OAuth2-Proxy Secret Alignment
When using OAuth2-proxy with Keycloak, the client secret must match in three places:

1. **Keycloak bootstrap script** - Fixed value in `modules/services/keycloak.nix`:
```nix
api_call POST "admin/realms/web/clients" \
  '{"clientId":"oauth2-proxy-glances","secret":"REDACTED_OAUTH2_CLIENT_SECRET",...}'
```

2. **oauth2-proxy-client-secret.age** - Must contain same value:
```bash
( cd /etc/nixos/secrets && echo "REDACTED_OAUTH2_CLIENT_SECRET" | nix run github:ryantm/agenix -- -e oauth2-proxy-client-secret.age -i ../keys/cmos-agenix.key )
```

3. **google-oauth-keycloak.age** - `OAUTH2_PROXY_CLIENT_SECRET` variable must match:
```
OAUTH2_PROXY_CLIENT_SECRET=REDACTED_OAUTH2_CLIENT_SECRET
```

**Troubleshooting**: If OAuth2-proxy shows "Invalid client or Invalid client credentials", verify all three values match using:
```bash
# Check secret file
cat /run/agenix/oauth2-proxy-client-secret

# Check environment file
grep OAUTH2_PROXY_CLIENT_SECRET /run/agenix/google-oauth-keycloak

# Query Keycloak for actual client secret
/nix/store/*/keycloak-*/bin/kcadm.sh get clients -r web -q clientId=oauth2-proxy-glances --fields secret
```

## Quick Operations

### Adding a New Secret (Shared)
```bash
# 1. Edit secrets/secrets.nix
"my-secret.age".publicKeys = allKeys;

# 2. Edit modules/core/agenix-secrets.nix  
my-secret = {
  file = ../../secrets/my-secret.age;
  mode = "0600";
  owner = "root";
  group = "root";
};

# 3. Encrypt (with preflight check)
test -f /etc/nixos/secrets/secrets.nix || { echo "Missing secrets.nix"; exit 1; }
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -e my-secret.age -i ../keys/cmos-agenix.key )

# 4. Use in config
config.age.secrets.my-secret.path
```

### Adding a New Secret (Host-Specific)
```bash
# Use single host key instead of allKeys:
"hostname-secret.age".publicKeys = [ hostnameKey ];
```

### Re-encrypting Secrets (Bulk)
```bash
# Preflight check
test -f /etc/nixos/secrets/secrets.nix || { echo "Missing secrets.nix"; exit 1; }
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -r -i ../keys/cmos-agenix.key )
```

### Testing Secret Decryption
```bash
# Test if current host can decrypt a secret:
test -f /etc/nixos/secrets/secrets.nix || { echo "Missing secrets.nix"; exit 1; }
( cd /etc/nixos/secrets && nix run github:ryantm/agenix -- -d secret-name.age -i ../keys/$(hostname)-agenix.key )
```

## Security Models

### Option 1: All Hosts Decrypt All Secrets (Simpler)
```nix
# All secrets use allKeys - easier management, less secure
"every-secret.age".publicKeys = allKeys;
```
**Pros**: No re-encryption when adding hosts, simpler operations  
**Cons**: Secrets available on hosts that don't need them

### Option 2: Granular Access Control (More Secure)  
```nix
# Shared secrets for common needs
"user-password.age".publicKeys = allKeys;
# Host-specific secrets for sensitive data
"cmos-ssh-key.age".publicKeys = [ cmosKey ];
```
**Pros**: Better security isolation  
**Cons**: Must re-encrypt when adding hosts

## Automated Secret Backups

**Script**: `/etc/nixos/scripts/backup-secrets.py`

Backs up all agenix secrets and keys to three destinations:

```bash
# Backup to 1Password (default)
python3 /etc/nixos/scripts/backup-secrets.py

# Backup to Google Drive (YubiKey-encrypted, uses rclone)
python3 /etc/nixos/scripts/backup-secrets.py --gdrive

# Backup to NAS (password-protected zip)
python3 /etc/nixos/scripts/backup-secrets.py --nas

# Preview without making changes
python3 /etc/nixos/scripts/backup-secrets.py --dry-run
```

**Backup Destinations**:

| Flag | Destination | Encryption | Notes |
|------|-------------|------------|-------|
| (none) | 1Password vault "nixos" | 1Password | Default, rate-limited |
| `--gdrive` | `gdrive:Backups/nixos/` | YubiKey age | Uses rclone, empties trash |
| `--nas` | `/mnt/syno/backups/nixos/` | Password zip | Timestamped files |

**How it works**:
- Automatically detects which host can decrypt each secret based on `secrets.nix`
- Decrypts locally on cmos, or via SSH to misc/jimmich for their exclusive secrets
- Google Drive backup uses rclone and automatically empties trash after replacing old backup

**1Password item naming**: Secrets stored as `nixos-secret-&lt;name>`, keys as `nixos-key-&lt;name>`

## Prevention Checklist

When making changes to the secret management system:

- [ ] All new `.age` files added to git before building
- [ ] Age keys have proper ownership (root:root) and permissions (600)
- [ ] SSH authorized keys include all deployment hosts
- [ ] Shared secrets re-encrypted when adding new hosts
- [ ] Test manual decryption before deploying
- [ ] Check `/run/agenix/` after deployment for successful decryption
- [ ] Run `backup-secrets.py` to backup new secrets to 1Password

For detailed information, workflows, and advanced troubleshooting, see `/etc/nixos/docs/secrets/agenix.md`.
