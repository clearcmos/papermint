## Model Overview

The system uses three specialized models, each optimized for specific tasks:

| Model | Purpose | Size | Applications |
|-------|---------|------|--------------|
| **qwen2.5-coder:7b-instruct** | Coding assistance | 4.7GB | smart-commit |
| **qwen2.5:7b-instruct** | General knowledge | 4.7GB | keeper, sage |
| **bge-m3** | Text embeddings | 1.1GB | keeper |

## Model Specifications

### qwen2.5-coder:7b-instruct

**Purpose**: Advanced coding assistance and code generation

**Specifications**:
- **Parameters**: 7.6B
- **Quantization**: Q4_K_M
- **Context Length**: 8192 tokens
- **VRAM Usage**: ~4.7GB
- **Specialization**: Code completion, debugging, documentation

**API Usage**:
```bash
curl -X POST http://localhost:11434/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5-coder:7b-instruct",
    "prompt": "Write a Python function to calculate factorial",
    "stream": false
  }'
```

**Applications**:
- **smart-commit**: Generates intelligent Git commit messages based on code changes
- **Code analysis**: Understands programming patterns and best practices
- **Documentation**: Generates code comments and documentation

**Performance Characteristics**:
- **Strengths**: Code understanding, multiple programming languages, efficient inference
- **Context handling**: Excellent for large code files and complex logic
- **Speed**: ~50-80 tokens/second on RX 6800 XT

### qwen2.5:7b-instruct

**Purpose**: General knowledge and conversational AI

**Specifications**:
- **Parameters**: 7.6B
- **Quantization**: Q4_K_M
- **Context Length**: 8192 tokens
- **VRAM Usage**: ~4.7GB
- **Specialization**: Natural language understanding, knowledge tasks

**API Usage**:
```bash
curl -X POST http://localhost:11434/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5:7b-instruct",
    "prompt": "Categorize this note: Meeting about Q4 budget planning",
    "stream": false
  }'
```

**Applications**:
- **keeper**: Note categorization and title generation
- **sage**: Linux system administration assistance
- **General queries**: Knowledge-based question answering

**Performance Characteristics**:
- **Strengths**: Broad knowledge base, multilingual support, instruction following
- **Context handling**: Excellent for complex reasoning tasks
- **Speed**: ~50-80 tokens/second on RX 6800 XT

### bge-m3

**Purpose**: Text embeddings for semantic similarity and search

**Specifications**:
- **Type**: Embedding model
- **Quantization**: FP16
- **Dimensions**: 1024-dimensional vectors
- **VRAM Usage**: ~1.1GB
- **Specialization**: Semantic understanding, text similarity

**API Usage**:
```bash
curl -X POST http://localhost:11434/api/embeddings \
  -H "Content-Type: application/json" \
  -d '{
    "model": "bge-m3",
    "prompt": "Sample text for embedding generation"
  }'
```

**Applications**:
- **keeper**: Semantic search within notes
- **Document similarity**: Finding related content
- **Clustering**: Grouping similar text content

**Performance Characteristics**:
- **Strengths**: High-quality embeddings, multilingual support, efficient processing
- **Processing speed**: ~100-200 embeddings/second
- **Memory efficient**: Low VRAM usage compared to language models

## Model Management

### Automatic Download

Models are automatically downloaded on first system startup:

```bash
# Check download status
systemctl status ollama-model-preload.service

# Manual model download
ollama pull qwen2.5-coder:7b-instruct
ollama pull qwen2.5:7b-instruct
ollama pull bge-m3
```

### Model Storage

```bash
# Default model location
/var/lib/ollama/models/

# Check model storage usage
du -sh /var/lib/ollama/models/

# List installed models
ollama list
```

### Model Loading Behavior

The system uses intelligent model loading:

1. **On-Demand Loading**: Models load when first requested
2. **Memory Limit**: Maximum 2 models in VRAM simultaneously
3. **LRU Eviction**: Least recently used models are unloaded
4. **Keep-Alive**: Models stay loaded for 3 minutes after last use

```bash
# Check currently loaded models
curl -s http://localhost:11434/api/ps | jq '.models[]'

# Force model loading
curl -X POST http://localhost:11434/api/generate \
  -d '{"model": "qwen2.5-coder:7b-instruct", "prompt": "test", "keep_alive": "10m"}'
```

## Performance Optimization

### VRAM Management

**Current Configuration** (Gaming-Optimized):
- **Maximum loaded**: 2 models (~9-10GB VRAM)
- **Reserved for gaming**: 6-7GB VRAM
- **Total VRAM**: 16GB (RX 6800 XT)

**Alternative Configurations**:

```bash
# Maximum AI performance (all models loaded)
# Edit /etc/nixos/modules/desktop/ollama.nix:
OLLAMA_MAX_LOADED_MODELS = "3"  # Uses ~11GB VRAM

# Maximum gaming compatibility (one model only)
OLLAMA_MAX_LOADED_MODELS = "1"  # Uses ~5GB VRAM
```

## Model Quality and Quantization

### Quantization Levels

**Q4_K_M (Used by language models)**:
- **Quality**: High (minimal loss from FP16)
- **Size reduction**: ~4x smaller than FP16
- **Performance**: Excellent balance of quality and speed
- **Memory usage**: Optimal for GPU inference

**FP16 (Used by bge-m3)**:
- **Quality**: Maximum (no quantization loss)
- **Size**: Larger than quantized versions
- **Performance**: Fast embedding generation
- **Use case**: Embeddings require high precision

### Quality Comparison

| Quantization | Model Size | Quality | Speed | VRAM Usage |
|--------------|------------|---------|--------|------------|
| **Q4_K_M** | 4.7GB | 98% of FP16 | Fast | Optimal |
| **Q5_K_M** | 5.7GB | 99% of FP16 | Fast | Higher |
| **Q8_0** | 7.2GB | 99.5% of FP16 | Medium | High |
| **FP16** | 14GB | 100% | Slow | Maximum |

Current choice (Q4_K_M) provides the best balance for gaming + AI workloads.

## Custom Model Integration

### Adding New Models

1. **Download model**:
```bash
ollama pull new-model:version
```

2. **Update preload service** (`/etc/nixos/modules/desktop/ollama.nix`):
```nix
MODELS=(
  "qwen2.5-coder:7b-instruct"
  "qwen2.5:7b-instruct"
  "bge-m3"
  "new-model:version"  # Add here
)
```

3. **Rebuild system**:
```bash
sudo nixos-rebuild switch --flake .#cmos
```

### Model Requirements

**For Language Models**:
- **Size**: &lt;8GB for gaming compatibility
- **Format**: GGUF quantized (Q4_K_M or Q5_K_M)
- **Context**: 8K+ tokens recommended
- **Architecture**: Compatible with Ollama

**For Embedding Models**:
- **Size**: &lt;2GB recommended
- **Dimensions**: 1024 or similar
- **Format**: GGUF or native Ollama format
- **Performance**: Fast inference required

## Troubleshooting Models

### Common Issues

**Model not loading**:
```bash
# Check available VRAM
rocm-smi --showmeminfo vram

# Check model exists
ollama list | grep model-name

# Force reload
curl -X POST http://localhost:11434/api/generate \
  -d '{"model": "model-name", "keep_alive": 0}'
```

**Slow performance**:
```bash
# Check GPU utilization
rocm-smi

# Monitor system resources
htop

# Check model size vs available VRAM
ollama show model-name
```

**Out of memory errors**:
```bash
# Reduce loaded models
# Edit OLLAMA_MAX_LOADED_MODELS in configuration

# Check for memory leaks
systemctl restart ollama.service
```
