## Overview

This system uses **Ollama** as the single AI service providing local large language model capabilities for all applications. The setup is optimized for both high-performance AI processing and gaming workloads on an AMD RX 6800 XT GPU system.

## Quick Start

```bash
# Check AI service status
systemctl status ollama.service

# List available models
curl -s http://localhost:11434/api/tags | jq '.models[].name'

# Test AI response
curl -X POST http://localhost:11434/api/generate \
  -H "Content-Type: application/json" \
  -d '{"model": "qwen2.5-coder:7b-instruct", "prompt": "Hello", "stream": false}'

# Validate full system
bash /etc/ollama-validate.sh
```

## Architecture

### Single Service Design
- **Service**: Ollama on port 11434
- **Models**: 3 specialized models for different tasks
- **GPU**: AMD RX 6800 XT with ROCm acceleration
- **Storage**: `/var/lib/ollama/models` (standard location)

### Model Allocation
- **qwen2.5-coder:7b-instruct**: Coding assistance (smart-commit)
- **qwen2.5:7b-instruct**: General knowledge tasks (keeper, sage)
- **bge-m3**: Text embeddings (keeper)

## Applications Using AI

### Smart-Commit
- **Model**: qwen2.5-coder:7b-instruct
- **Purpose**: AI-powered Git commit message generation
- **Endpoint**: `http://localhost:11434/api/generate`
- **Status**: ✅ Working

### Keeper
- **Models**: qwen2.5:7b-instruct (categorization), bge-m3 (embeddings)
- **Purpose**: Google Keep note management and AI categorization
- **Endpoint**: `http://localhost:11434/api/generate` & `/api/embeddings`
- **Status**: ✅ Ready for testing

### Sage
- **Model**: qwen2.5:7b-instruct
- **Purpose**: AI-powered Linux system management assistant
- **Endpoint**: `http://localhost:11434/api/generate`
- **Status**: ✅ Ready for testing

### Audio Separator (UVR5)
- **Technology**: PyTorch ROCm + MDX-Net/Demucs models
- **Purpose**: GPU-accelerated vocal/instrumental audio separation
- **Commands**: `uvr`, `uvr-models`, `uvr-help`
- **Status**: ✅ Working

## Resource Usage

### VRAM Allocation (16GB Total)
- **AI Models**: 9-10GB maximum (2 models loaded)
- **WoW/Gaming**: 6-7GB reserved
- **System**: 1GB for desktop/apps

### Performance Specifications
- **Parallel Requests**: 6 per model
- **Queue Size**: 1024 requests
- **Context Length**: 8192 tokens
- **Model Unload Time**: 3 minutes after last use
