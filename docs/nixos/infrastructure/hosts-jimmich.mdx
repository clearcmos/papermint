## 🎯 Overview

**jimmich** is the secondary server in the infrastructure, dedicated to file services, backup storage, and providing redundancy for critical data. This host runs a minimal server configuration focused on reliability and storage capacity.

**Role**: Secondary Server, File Services, Backup Target, Storage Host

---

## 💻 Hardware Specifications

### System Information
- **Hostname**: `jimmich.home.arpa`
- **Platform**: Standard server hardware
- **Architecture**: x86_64
- **Boot Loader**: Default (configured in hardware-configuration.nix)
- **Form Factor**: Server/Desktop

### Storage
- **Configuration**: Managed in `hardware-configuration.nix`
- **Purpose**: High-capacity storage for files and backups
- **File System**: ext4 (typical)

### Network Hardware
- **Interface**: `enp8s0` (standard Ethernet adapter)
- **Connection**: Gigabit Ethernet

## 🌐 Network Configuration

### Network Interface
- **Interface**: `enp8s0`
- **IP Address**: `192.168.1.10/24` (static)
- **Gateway**: `192.168.1.1`
- **DNS**: `192.168.1.1`

**Firewall Profile**: Server-only (no desktop services)

### Samba Share
**Module**: `modules/core/samba-server.nix` (from core)

**Path**: `/home/nicholas/`
**Access**: Password-protected (nicholas user)

**Purpose**:
- Cross-host file sharing
- Backup storage for other hosts
- Media file storage
- General file server

**Clients**:
- cmos (desktop workstation)
- misc (primary server)
- nixvm (development VM)

**Mounting on other hosts**:
```bash
# Example mount on cmos
mount -t cifs //jimmich.home.arpa/nicholas-home /mnt/jimmich-home \
  -o credentials=/run/agenix/smb-credentials,uid=1000,gid=100
```

## 🧪 Troubleshooting

### SSH Access Issues

**Can't connect via SSH**:
```bash
# From another host
ping jimmich.home.arpa

# Check SSH service
ssh root@192.168.1.10 'systemctl status sshd'
```

**OAuth authentication problems**:
```bash
# Check OAuth service
ssh root@jimmich.home.arpa 'systemctl status keycloak-ssh-agent'
journalctl -u keycloak-ssh-agent -f
```

### Samba Issues

**Can't access Samba shares**:
```bash
# Test from another host
smbclient -L jimmich.home.arpa -U nicholas

# Check Samba status
ssh root@jimmich.home.arpa 'systemctl status smbd nmbd'
```

**Check Samba configuration**:
```bash
ssh root@jimmich.home.arpa 'testparm -s'
```

**View Samba logs**:
```bash
ssh root@jimmich.home.arpa 'journalctl -u smbd -f'
```

### Storage Issues

**Check disk space**:
```bash
ssh root@jimmich.home.arpa 'df -h'
```

**Check storage health**:
```bash
ssh root@jimmich.home.arpa 'smartctl -a /dev/sda'
```

### Network Issues

**Check network configuration**:
```bash
ssh root@jimmich.home.arpa 'ip addr show enp8s0'
ssh root@jimmich.home.arpa 'ip route'
```

**Test connectivity**:
```bash
# From jimmich to gateway
ssh root@jimmich.home.arpa 'ping -c 3 192.168.1.1'

# From jimmich to cmos
ssh root@jimmich.home.arpa 'ping -c 3 192.168.1.2'
```

## 🔄 Backup Strategy

### As Potential Backup Target
**Note**: Currently, cmos uses remote backup to `offsite.bedrosn.com` instead of jimmich.

**Current Backup Architecture**:
- **cmos**: Local Borg (`/mnt/data/backups/cmos/borg`) + Remote Borg (`offsite.bedrosn.com`)
- **jimmich**: Not currently receiving automated backups from other hosts

**Potential Future Use**:
- Could serve as local network backup target via Samba
- Useful for large file storage and secondary backup location
- Sufficient storage capacity for backup hosting

See **[storage-backup.md](../storage/backup.md)** for complete backup documentation.

### Manual Backup Copy (if needed)
```bash
# From cmos (via Samba)
mount -t cifs //jimmich.home.arpa/nicholas-home /mnt/jimmich \
  -o credentials=/run/agenix/smb-credentials

# Manual backup copy to jimmich
rsync -av /mnt/data/backups/ /mnt/jimmich/backups/
```

## 💡 Usage Examples

### Accessing Files from Other Hosts

**Mount Samba share**:
```bash
# From cmos
sudo mkdir -p /mnt/jimmich-home
sudo mount -t cifs //jimmich.home.arpa/nicholas-home /mnt/jimmich-home \
  -o credentials=/run/agenix/smb-credentials,uid=1000,gid=100
```

**Copy files to jimmich**:
```bash
# From any host
scp -r /local/files/ root@jimmich.home.arpa:/home/nicholas/storage/
```

**SSH access for management**:
```bash
# From any host
ssh root@jimmich.home.arpa

# Run commands remotely
ssh root@jimmich.home.arpa 'df -h'
ssh root@jimmich.home.arpa 'systemctl status smbd'
```
