## 🎯 Overview

**misc** is the primary server in the infrastructure, hosting all public-facing web services, centralized authentication, reverse proxy, and monitoring systems. This host is the gateway for external access and provides essential services for the entire network.

**Role**: Primary Server, Web Services Host, Authentication Provider, Reverse Proxy

---

## 💻 Hardware Specifications

### System Information
- **Hostname**: `misc.home.arpa`
- **Platform**: Mac Mini (Intel)
- **Architecture**: x86_64
- **Boot Loader**: systemd-boot with EFI
- **Form Factor**: Compact server (Mac Mini)

### Storage
- **Root**: NVMe SSD
- **Partition Layout**:
  - `/boot` - 1GB FAT32 (ESP)
  - `/` - Primary partition (ext4)
  - Swap - 9GB (linux-swap)

### Network Hardware
- **Interface**: `enp3s0f0` (dual-port server-grade NIC)
- **Connection**: Gigabit Ethernet

## 🌐 Web Services

### Traefik Reverse Proxy
**Module**: `modules/services/traefik.nix`
**Ports**: 80 (HTTP), 443 (HTTPS)
**Features**:
- Automatic HTTPS via Let's Encrypt
- Dynamic service discovery
- Middleware support (compression, headers, auth)
- Dashboard with metrics

**Public Services**:
- `https://bedrosn.com/*` - Various web services
- Automatic HTTP → HTTPS redirect
- Rate limiting and security headers

### Keycloak Identity Provider
**Module**: `modules/services/keycloak.nix`
**Port**: 8089 (localhost only)
**Database**: PostgreSQL
**Purpose**:
- SSH OAuth device flow authentication
- Central identity management
- SSO integration capabilities

**Realms**:
- SSH realm - SSH device flow authentication
- Web realm - Web service authentication (if configured)

**Access**: Via Traefik reverse proxy only

### PostgreSQL Database
**Module**: `modules/services/postgresql.nix`
**Purpose**: Shared database for services
**Databases**:
- Keycloak (identity provider)

**Access**: Localhost only via Unix sockets

## 🛡️ Security Services

### OAuth2-Proxy Services
**Modules**: `modules/services/oauth2-proxy-*.nix`
**Purpose**: Authentication proxies for web services
**Ports**: 4180-4189 (various services)
**Features**:
- Integration with Keycloak OIDC
- Group-based authorization
- Session management (4h with 15min refresh)
- Cookie-based authentication

**Protected Services**:
- Glances (port 4180)
- Traefik Dashboard (port 4181)
- Security Dashboard (port 4182)
- Filebrowser (port 4184)
- Cockpit instances (ports 4185-4188)
- NixDocs (port 4189)

### Fail2ban
**Module**: `modules/services/fail2ban.nix`
**Purpose**: Intrusion prevention and brute-force protection
**Features**:
- 8 jails protecting Traefik services
- Dynamic WAN IP whitelisting (every 30 min)
- Ban duration: 1 hour
- Max retry: 5 attempts

### Security Dashboard
**Module**: `modules/services/security-dashboard.nix`
**Port**: 8091 (backend), 4182 (OAuth2-proxy)
**Purpose**: Centralized security posture monitoring
**Features**:
- Real-time service status
- Security header analysis with scoring
- User access control visibility
- Background caching (5min refresh)

**Access**: Via OAuth2-proxy (port 4182)

**Module**: `modules/services/sabnzbd.nix`

**Configuration**:
```nix

  port = 8081;      # Backend
  proxyPort = 8090; # Docker nginx proxy
```

## 🚀 Deployment

### Bootstrap Deployment
**Method**: Automated ISO Installation
**Documentation**: `docs/operations-bootstrap-automated-iso.md`

**Quick Deploy**:
```bash
# After bootstrap ISO installation
ssh root@192.168.1.5 'mkdir -p /etc/nixos/keys'
scp /etc/nixos/keys/misc-agenix.key root@192.168.1.5:/etc/nixos/keys/
nixos-rebuild switch --flake .#misc --target-host misc.home.arpa
```

### Remote Deployment
```bash
# From cmos or any host
nixos-rebuild switch --flake .#misc --target-host misc.home.arpa
```

### Service Management
```bash
# Restart specific service
ssh root@misc.home.arpa 'systemctl restart traefik'

# Check service status
ssh root@misc.home.arpa 'systemctl status keycloak'

# View logs
ssh root@misc.home.arpa 'journalctl -u security-dashboard -n 50'
```

## 🧪 Troubleshooting

### Traefik Issues

**Check Traefik status**:
```bash
systemctl status traefik
journalctl -u traefik -f
```

**View Traefik dashboard**:
```bash
# Access via configured Traefik dashboard route
curl http://localhost:8080/dashboard/
```

**Certificate problems**:
```bash
# Check Let's Encrypt certificates
cat /var/lib/traefik/acme.json | jq '.[] | keys'
```

### Keycloak Issues

**Check Keycloak status**:
```bash
systemctl status keycloak
journalctl -u keycloak -f
```

**Database connection**:
```bash
# Verify PostgreSQL is running
systemctl status postgresql

# Check Keycloak database
sudo -u postgres psql -c "\l" | grep keycloak
```

### OAuth2-Proxy Issues

**Check OAuth2-proxy services**:
```bash
systemctl status 'oauth2-proxy-*'
journalctl -u oauth2-proxy-glances -f
journalctl -u oauth2-proxy-security-dashboard -f
```

**Test authentication**:
```bash
curl -I https://security.bedrosn.com
# Should redirect to Keycloak if not authenticated
```

### DNS Issues

**Check Cloudflare DNS service**:
```bash
systemctl status cloudflare-dns-upsert
journalctl -u cloudflare-dns-upsert -n 50
```

**Manual DNS update**:
```bash
# Trigger DNS update
systemctl start cloudflare-dns-upsert
```

### PrivateBin Issues

**Check Docker containers**:
```bash
docker ps | grep privatebin
docker logs privatebin-nginx
```

**Restart PrivateBin**:
```bash
systemctl restart privatebin-custom
```
