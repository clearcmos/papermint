## ☁️ Cloud Storage Integration

### 📤 Google Drive Integration

**[Google Drive Configuration](google-drive.md)**:
- **Access Method**: KDE Online Accounts with KIO worker
- **Protocol**: `gdrive:/` virtual filesystem in Dolphin
- **Sync Type**: On-demand streaming (not local sync)
- **Authentication**: OAuth2 via signond + signon-ui

**Access in Dolphin**:
```
gdrive:/                    # Root of Google Drive
gdrive:/google11/           # Account-specific access
```

**Key Features**:
- **Native Integration**: Direct access in KDE Dolphin file manager
- **On-Demand Access**: Files streamed when accessed (not cached locally)
- **Drag & Drop**: Full file operations support
- **No Local Storage**: Files remain in cloud until explicitly downloaded

**Security Considerations**:
- **OAuth2 Tokens**: GNOME credentials for Google compatibility
- **KWallet Storage**: Credentials stored securely in KWallet
- **Scoped Access**: Limited to Drive API permissions
- **No Local Sync**: Reduces data exposure risk

### 🔄 Access Commands

**Dolphin File Manager**:
```bash
# Open Google Drive in Dolphin
dolphin gdrive:/

# Or use the KDE address bar
# Type: gdrive:/
```

**Account Management**:
```bash
# Manage accounts via System Settings
systemsettings kcm_kaccounts
```

**Note**: Google Drive via KIO is streaming-only. For offline access or local sync, consider using `rclone` manually.

## 🛡️ Backup Strategy

### 📋 Backup Architecture

**[Comprehensive Backup Strategy](../storage/backup.md)**:

**Dual-Destination Borg Backup**:
- **Local**: `/mnt/data/backups/cmos/borg` (immediate access)
- **Remote**: `nicholas@offsite.bedrosn.com:/mnt/data/cmos/borg` (off-site protection)
- **Encryption**: repokey-blake2 with agenix-managed passphrase
- **Deduplication**: Borg's intelligent chunking
- **Compression**: zstd level 3

### 🔄 Backup Types & Schedules

**Automated Borg Backups (cmos)**:
- **Local Backup**: Daily at 5:00 AM EDT
- **Remote Backup**: Triggered automatically after local success
- **Retention**: 14 daily backups (auto-pruned)
- **Monitoring**: Log files and systemd journal

**Backed Up Paths**:
- **System**: `/etc/nixos` (complete configuration)
- **User Data**: Documents, Pictures, Videos, git repos
- **Configurations**: KDE settings, SSH keys, bash history
- **Root Data**: GPG keys, SSH keys

**Manual Backups**:
- **Borg Manual**: `backup-cmos-manual-borg` (on-demand Borg)
- **rsync Manual**: `backup-cmos-manual` (timestamped snapshots)

### 🎯 Backup Verification

**Integrity Checks**:
```bash
# Verify repository integrity
borg check /mnt/data/backups/cmos/borg

# Verify specific archive
borg check /mnt/data/backups/cmos/borg::cmos-20251014-050001

# List all archives
export BORG_PASSPHRASE=$(sudo cat /run/agenix/borg-backup-passphrase)
borg list /mnt/data/backups/cmos/borg
```

**Recovery Testing**:
- **Monthly**: Test restore of random files
- **Quarterly**: Full home directory restore test
- **Annually**: Complete disaster recovery simulation
