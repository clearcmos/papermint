## The Problem

Managing 1,000+ users across Okta involves repetitive, error-prone manual work: access investigations, group membership audits, bulk provisioning/deprovisioning, MFA factor exports, compliance evidence collection, and cross-system synchronization. Each task requires navigating the Okta admin console, running API calls manually, and cross-referencing data across systems.

## What Was Built

A library of 35 purpose-built automation tools (~25,600 lines of Python) covering the full identity lifecycle.

### Investigation & Forensics

| Tool | Purpose |
|------|---------|
| `investigation-toolkit` | Forensic investigation suite — user lookups, system log analysis, group membership tracing, access timeline reconstruction. Integrated into the `it-support-ticket` Claude Code skill for AI-assisted triage. |
| `get-user-access` | Full access report for a user — all apps, groups, roles |
| `get-user-details` | Comprehensive user profile dump |
| `get-user-devices` | Device inventory for a specific user |
| `check-provisioning-logs` | Provisioning event analysis for troubleshooting app assignment failures |
| `verify-okta-status` | Bulk user status verification |

## Architecture Patterns

All tools follow consistent patterns:
- **1Password credential retrieval** with environment variable fallback
- **Rate limit handling** with exponential backoff
- **Pagination** through all Okta API responses (critical — the org has ~2,000 group rules)
- **CSV/JSON output** for downstream processing
- **Progress bars** (tqdm) for long-running operations
- **Dry-run modes** where destructive operations are involved

## Impact

- Investigation toolkit reduces L3 ticket investigation from 30-60 minutes to 5-10 minutes
- Bulk operations that previously took hours of manual console work execute in minutes
- Compliance exports that required manual screenshot collection are now automated with full data capture
- Cross-system audits (Okta vs. Kandji, Okta vs. O365, Okta vs. Rippling) catch access drift automatically
- The entire suite is reusable — any ISS team member can run these tools without understanding the Okta API
